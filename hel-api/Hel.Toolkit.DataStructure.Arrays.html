DynamicArray<TDataType>

Dynamically sized array.

When this array's size is reached it automatically expands. The main difference between this and a list is the way data is stored and assigned. A DynamicArray is JUST an array. It does not sort your data in any way. This means a DynamicArray has the same performance as a regular array compared to a List. In exchange for higher performance (Insert for example) this data structure is more difficult to use and manipulate than a list.

Use this for performance-critical logic, or when you need your data to maintain its index as well as fast insert times.

ManagedArray<TDataType>

Automatically managed and assigned array. A ManagedArray is, at its core, just an array and as such it has very similar performance. Indexes are never sorted or modified. When an object is removed from the Array, that index is returned to the FreeId queue, allowing it to be recycled without disturbing the other indexes.

This is important for data that needs to be retrieved by index key
